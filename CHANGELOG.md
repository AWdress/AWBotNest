# 更新日志

本项目所有重要变更都记录在此。版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## v1.0.3

- 修复多个插件监听同类消息时互相"吃消息"：平台为每个插件分配独立的 group 区间，不同插件都能收到同一条消息，互不抢占；插件内部仍可用相对 group 排优先级。
- 新增 `ctx.StopPropagation`：插件可在处理器内主动中断消息传播，阻止后续插件再处理这条消息。
- 支持插件自带图标：`__plugin__` 新增可选 `icon` 字段（图标 URL），已安装插件卡片不再一律回退平台 logo，可显示插件自己的图标。
- 修复多账号「只勾选一个账号、两个账号都在发消息」：`ctx.user` / `ctx.user_apps` 此前忽略账号范围、返回全部已连接账号，导致主动遍历账号发消息的插件作用到未勾选的账号。现与处理器挂载共用同一过滤，账号范围一致生效。
- 插件仓库轮询更新已安装插件后，若该插件正在运行则自动热重载，新代码立即生效（此前需手动「重载」或重启）；未运行的插件只更新文件、不自动启用。
- 修复「刷新市场看不到新版本、重启才更新」：`raw.githubusercontent.com` 上的 manifest 受 Fastly CDN 缓存（约 5 分钟）影响，刚推送的版本在 TTL 内刷新仍读到旧值。现读取 manifest、探测文件夹插件、下载文件均带缓存绕过参数与 no-cache 头，刷新即取最新。
- 取消插件下载的单文件大小上限（原 8MB），插件可包含较大资源文件。
- 插件可声明第三方依赖：`__plugin__` 新增可选 `requirements`（PEP 508 列表），启用时平台统一代装（插件不自调 pip）。装前对照当前环境检测冲突——已满足跳过、缺失安装、版本冲突则拒绝启用并提示原因（单进程同一包无法多版本共存）；依赖声明经规范化后传 pip，杜绝参数注入。
- 插件依赖安装走平台代理（复用 `proxy_set`，墙内环境也能装），并限制重试/超时，连不上 pypi 时快速失败，不再长时间占用安装锁拖住其它插件启停。
- 新增 `PIP_INDEX_URL` 设置（设置→运行代理）：插件装依赖默认走清华镜像，境内直连不经墙，开箱可用；留空则回退官方 pypi（配了代理则用代理出墙）。镜像源直连、不再套代理，规避墙内到 pypi 的 TLS 干扰。

## v1.0.2

- 插件市场访问 GitHub 走平台代理，墙内环境也能正常浏览/安装插件。
- AI 接口（OpenAI 等）走平台代理，墙内可正常调用。
- 新增「重启平台」按钮（侧边栏）与重启接口，配置改动后可一键重启生效。
- 支持手机 PWA：可"添加到主屏幕"独立运行，带应用图标与离线兜底；手机版改用底部标签栏导航 + 卡片布局，适配小屏。
- 网页图标（favicon）改用平台 logo；未配置凭据时 Bot 启动只提示一行，不再刷错误堆栈。
- 活跃统计改为只计插件真正发出的动作（发消息/回复/编辑），空闲不再虚涨；活跃图例固定高度多列排布，插件多了不再无限拉长。
- 修复插件启用状态会"自己关闭"：加载失败不再抹掉启用意图，仅手动停用才真正关闭。
- 插件卡片仓库来源只显示 owner/repo（不再显示完整链接）。
- 登录页改为版权 + 版本号页脚；默认账号密码改在后台日志显示，登录页不再暴露。
- 修复重启/更新镜像后插件全部变「未启用」：进程退出时的卸载不再持久化停用状态，重启后按原启用状态自动恢复。
- 修复 Bot 未连接时插件 handler 被挂到无效连接（启用了却收不到消息）；Bot 启动失败清理残留实例。
- 修复轮询会覆盖本地自传/导入的同名插件：只自动更新「确实从仓库下载过」的插件，本地改动不再被冲掉。
- 插件下载改为全部成功才落盘，文件夹插件中途失败不再留半截文件。
- 安全加固：登录口令恒定时间比较、GitHub raw 源精确校验主机（防 SSRF）、下载文件大小封顶、路径校验统一最严。
- 安全：强制首次改密——仍使用默认密码时，登录后必须先改密才能进入控制台，且上传/导入/启用/设置等高危操作在改密前一律拒绝（堵住默认密码被接管的风险）。

## v1.0.0

首个正式版本。

- 平台内核 + 单文件插件：一个功能 = 一个 `.py` 文件，上传即用，真热插拔（启用/停用不重启进程）。
- 插件市场：内置官方仓库 `AWdress/AWBotNest-Plugins`，可配置多个 GitHub 仓库，浏览并一键安装插件（只下载不自动启用）。
- 多账号：网页登录 Telegram 账号，可同时管理多个；插件支持按账号选择生效范围，不同号可开不同插件。
- Web 控制台：插件管理、账号管理、运行日志、系统状态、系统设置，全在深色界面中操作。
- 通知中心：插件通过 `ctx.notify` 提交，平台统一分类、标注账号、推送给平台主人。
- 用户名+密码登录（PBKDF2 哈希 + 无状态 HMAC 令牌）。
