# 更新日志

本项目所有重要变更都记录在此。版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## v1.0.2

- 插件市场访问 GitHub 走平台代理，墙内环境也能正常浏览/安装插件。
- AI 接口（OpenAI 等）走平台代理，墙内可正常调用。
- 新增「重启平台」按钮（侧边栏）与重启接口，配置改动后可一键重启生效。
- 支持手机 PWA：可"添加到主屏幕"独立运行，带应用图标与离线兜底；手机版改用底部标签栏导航 + 卡片布局，适配小屏。
- 网页图标（favicon）改用平台 logo；未配置凭据时 Bot 启动只提示一行，不再刷错误堆栈。
- 活跃统计改为只计插件真正发出的动作（发消息/回复/编辑），空闲不再虚涨；活跃图例固定高度多列排布，插件多了不再无限拉长。
- 修复插件启用状态会"自己关闭"：加载失败不再抹掉启用意图，仅手动停用才真正关闭。
- 插件卡片仓库来源只显示 owner/repo（不再显示完整链接）。
- 登录页改为版权 + 版本号页脚；默认账号密码改在后台日志显示，登录页不再暴露。
- 修复重启/更新镜像后插件全部变「未启用」：进程退出时的卸载不再持久化停用状态，重启后按原启用状态自动恢复。
- 修复 Bot 未连接时插件 handler 被挂到无效连接（启用了却收不到消息）；Bot 启动失败清理残留实例。
- 修复轮询会覆盖本地自传/导入的同名插件：只自动更新「确实从仓库下载过」的插件，本地改动不再被冲掉。
- 插件下载改为全部成功才落盘，文件夹插件中途失败不再留半截文件。
- 安全加固：登录口令恒定时间比较、GitHub raw 源精确校验主机（防 SSRF）、下载文件大小封顶、路径校验统一最严、容器改用非 root 运行。
- 安全：强制首次改密——仍使用默认密码时，登录后必须先改密才能进入控制台，且上传/导入/启用/设置等高危操作在改密前一律拒绝（堵住默认密码被接管的风险）。

## v1.0.1

- 全项目移除 emoji（README、文档、源码注释、日志、通知），界面与输出更简洁。
- 通知中心改用纯文字级别标签（【通知】【成功】【警告】【错误】），不再用图标。
- 运行日志去掉文件名与来源列，只显示「时间 · 级别 · 内容」，插件日志保留 [插件id] 前缀。

## v1.0.0

首个正式版本。

- 平台内核 + 单文件插件：一个功能 = 一个 `.py` 文件，上传即用，真热插拔（启用/停用不重启进程）。
- 插件市场：内置官方仓库 `AWdress/AWBotNest-Plugins`，可配置多个 GitHub 仓库，浏览并一键安装插件（只下载不自动启用）。
- 多账号：网页登录 Telegram 账号，可同时管理多个；插件支持按账号选择生效范围，不同号可开不同插件。
- Web 控制台：插件管理、账号管理、运行日志、系统状态、系统设置，全在深色界面中操作。
- 通知中心：插件通过 `ctx.notify` 提交，平台统一分类、标注账号、推送给平台主人。
- 用户名+密码登录（PBKDF2 哈希 + 无状态 HMAC 令牌）。
