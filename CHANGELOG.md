# 更新日志

本项目所有重要变更都记录在此。版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## v1.0.9

这一版主要是让运行日志更好读。

- 日志里的插件名显示中文名字：以前日志前缀是插件的英文代号（比如 `awpulse`），现在直接显示插件的中文名（比如「AWPulse 色花堂助手」），一眼就知道是哪个插件在动，不用再对着代号猜。插件启用、停用、删除、下载更新、装依赖等场景都统一显示中文名。
- 顺手把一批日志措辞改成大白话：像 `user_app`、`OSError`、`WebUI` 这类看不懂的术语，分别换成了「用户账号」「端口可能被占用」「网页控制台」等说法，非技术用户也能看明白日志在说什么。

## v1.0.8

这一版给插件作者两样大件：可以用 Vue 自己写插件的配置界面，以及平台内置了浏览器能力。

- 插件可以自带配置界面：除了平台按声明自动生成的表单，复杂插件现在可以用 Vue 写一个自己的配置页面（图表、可视化、复杂交互都行），平台在打开「配置」时直接加载它。普通插件继续用原来的自动表单，不受影响。
- 插件可以注册自己的接口：自带界面的插件能声明几个后端接口给自己的前端调用（读写数据、触发动作等），走管理员登录鉴权，外部访问不到。
- 插件前端更安全：插件的界面文件只有登录后的会话才能加载，未登录的人拿不到。
- 平台内置浏览器：插件需要抓渲染后的网页、过反爬 / 指纹检测时，直接用平台提供的浏览器能力即可，不用自己装。优先用停用浏览器 CloakBrowser（能过 Cloudflare / 指纹检测），不可用时自动回退 Playwright，插件无感。为不让镜像变大，浏览器内核不打进镜像，插件首次用到时才自动下载并缓存（随数据目录持久化）；不用浏览器的部署零负担。
- 附带一个「Vue 模式」插件模板，照着改就能上手。

## v1.0.7

这一版主要是让插件的「配置页」更好用，给插件作者多了几种现成的配置控件，管理员填配置也更省心。

- 配置项新增「列表」控件：可以一行行地增删条目，每行是一组小字段（比如一条条规则），不用再把多项内容硬塞进一个输入框。
- 配置项新增「会话选择器」：需要选群 / 频道 / 会话的地方，直接从下拉列表里挑，不用再手动去找并填 -100 开头的一长串 ID；没登录账号时也能手填 ID 兜底。
- 配置项新增「动作按钮」：插件可以在配置页放个按钮，点一下当场执行（比如「测试连接」「立即运行一次」），结果直接弹提示，不用干等。
- 配置项新增「只读说明 / 状态」：可以在配置页显示一段固定说明，或展示插件写回来的状态（比如「上次运行时间」「已处理条数」）。
- 配置项支持「必填 / 数值范围」校验：漏填必填项或数字超出范围时，保存会被拦下并标红提示，避免配错了却不知道。
- 插件能力增强：插件可以把结果写回自己的配置（用于持久化状态或回填到上面的「状态」显示），也能一行代码把消息里的图片 / 文件下载到自己的目录。
- 修掉了会话选择器「没选却显示一个 0」、必填校验对会话选择漏判等小问题。

## v1.0.6

这一版加了 Webhook：外部服务可以通过一个专属网址，主动触发平台或插件。密钥全平台统一——在「系统设置 → 通知」里点一下生成，平台和所有插件共用同一个。

- 给平台用：外部程序往平台的接收网址发一条数据，就能把内容当通知推送给管理员。
- 给插件用：插件声明自己需要 Webhook 后，就能在「插件 → 配置」弹窗里看到自己专属的接收网址（每个插件网址不同、密钥共用）；外部事件发过来时插件能收到并处理，还能回一段结果。
- 网址带密钥校验，没生成密钥就等于关闭；密钥旁有「随机」按钮一键生成。插件停用或重载后，它的接收地址会自动失效。

## v1.0.5

这一版支持多个通知机器人。

- 可以在「系统设置 → 通知」里配置多个 Bot（一个默认的，加若干额外的），并给每个插件单独指定通知发到哪个 Bot；不配的话全走原来那个 Bot，老部署不用改。
- 这个选择对插件发通知、插件用机器人发消息都一致生效。删掉某个 Bot 时，原本指向它的插件会自动回到默认 Bot。
- 新增或删除 Bot 需要重启平台才能连上 / 断开（和改凭据一样）；插件选哪个 Bot 是即时生效的。
- 把 BOT_TOKEN 从「Telegram 凭据」页挪到了「通知」页，和额外 Bot 放一起管理。
- 用词统一：通知里的「主人 / 所有者」都改叫「管理员」，和登录控制台的叫法一致。

## v1.0.4

这一版让插件联网自动走平台代理。

- 开启代理后，插件访问外网会自动经过代理，墙内环境不用再给每个插件单独配；访问本机地址（localhost）不走代理。
- 改了代理设置即时生效，新启动或重载的插件立刻用上新代理（已经建立的长连接仍需重启）。
- 把代理相关逻辑统一收拢到一处，Telegram 连接、装依赖、AI、GitHub 导入都共用，不再各写一套。

## v1.0.3

- 修复多个插件同时监听同类消息时互相「抢消息」：现在每个插件都能独立收到同一条消息，互不干扰；插件内部仍可自己排处理顺序。
- 插件可以主动「拦截」一条消息，处理完就不让后面的插件再处理它。
- 插件可以自带图标：已安装的插件卡片会显示插件自己的图标，不再一律用平台 logo。
- 修复「只勾选一个账号，结果两个账号都在动」：之前插件主动发消息时会忽略账号范围，现在严格按勾选的账号来。
- 插件仓库轮询更新后，如果该插件正在运行会自动重载让新代码生效（以前要手动重载或重启）；没在运行的只更新文件、不自动开启。
- 修复「刷新市场看不到新版本、要重启才更新」：这是 GitHub 缓存导致的，现在读取时都绕过缓存，刷新就能看到最新版。
- 取消了插件单个文件的大小限制（原来 8MB），插件可以带较大的资源文件。
- 插件可以声明需要的第三方库，启用时平台统一帮它安装（插件不自己装）；装之前会检查冲突——已经有的跳过、缺的装上、版本打架就拒绝启用并说明原因。
- 装插件依赖也走平台代理（墙内也能装），并限制了重试和超时，连不上就快速失败，不会长时间卡住其它插件。
- 新增「PIP 镜像源」设置（设置 → 运行代理）：默认走清华镜像，境内直连不用翻墙，开箱即用；留空则用官方源。
- 插件依赖改装到数据目录里（而不是程序目录），容器重建或换新镜像后已装的依赖不会丢，不用重装。

## v1.0.2

- 插件市场访问 GitHub 走平台代理，墙内也能正常浏览和安装插件。
- AI 接口（OpenAI 等）走平台代理，墙内可正常调用。
- 侧边栏新增「重启平台」按钮和接口，改完配置可以一键重启生效。
- 支持手机 PWA：可以「添加到主屏幕」像 App 一样独立运行，带图标和离线兜底；手机上改成底部标签栏 + 卡片布局，适配小屏。
- 网页图标改用平台 logo；没配置凭据时机器人启动只提示一行，不再刷一堆报错。
- 活跃统计改成只算插件真正发出的动作（发消息 / 回复 / 编辑），空闲时不再虚涨；图例排布也优化了，插件多了不会无限拉长。
- 修复插件启用状态会「自己关掉」：加载失败不再当成停用，只有手动停用才真正关闭。
- 插件卡片的仓库来源只显示 owner/repo，不再显示完整链接。
- 登录页页脚改成版权 + 版本号；默认账号密码改到后台日志里显示，登录页不再暴露。
- 修复重启或更新镜像后插件全变「未启用」：现在会按原来的启用状态自动恢复。
- 修复机器人没连上时插件被挂到无效连接（开了却收不到消息）；机器人启动失败会清理残留。
- 修复轮询会覆盖本地自己上传 / 导入的同名插件：只自动更新「确实从仓库下载过」的，本地改动不再被冲掉。
- 插件下载改成全部成功才落盘，文件夹插件中途失败不会留下半截文件。
- 安全加固：登录口令用恒定时间比较、严格校验 GitHub 下载来源（防伪造地址）、限制下载文件大小、统一最严的路径校验。
- 安全：强制首次改密——还在用默认密码时，登录后必须先改密才能进控制台，且改密前上传 / 导入 / 启用 / 设置等高危操作一律拒绝（堵住默认密码被人接管的风险）。

- 通知中心改用纯文字级别标签（【通知】【成功】【警告】【错误】），不再用图标。
- 运行日志去掉文件名和来源列，只显示「时间 · 级别 · 内容」，插件日志保留 [插件id] 前缀。

## v1.0.0

首个正式版本。

- 平台内核 + 单文件插件：一个功能 = 一个 `.py` 文件，上传即用，真热插拔（启用 / 停用不用重启程序）。
- 插件市场：内置官方仓库，也可以自己加多个 GitHub 仓库，浏览并一键安装插件（只下载，不自动开启）。
- 多账号：网页登录 Telegram 账号，可同时管理多个；插件能按账号选择在哪些号上生效，不同号可以开不同插件。
- Web 控制台：插件管理、账号管理、运行日志、系统状态、系统设置，都在深色界面里操作。
- 通知中心：插件统一提交通知，平台负责分类、标注是哪个账号、推送给管理员。
- 用户名 + 密码登录（密码加密存储，登录令牌无需服务端保存状态）。
