# 更新日志

这里记录每个版本给用户带来的新功能、改进和问题修复。

## v1.1.0.5

这一版让多个通知 Bot 更方便管理，新增、切换和修改后不用重启整个平台。

- 内置 Bot 的名称现在可以修改，多个 Bot 中的任意一个都可以设为默认 Bot。
- 新增、删除、改名、更换 Token 或切换默认 Bot 后，保存设置就会立即生效。
- 更换 Token 时会先确认新 Bot 能正常连接，再停用旧连接；如果新 Token 有问题，原来的 Bot 会继续工作。
- 删除当前默认 Bot 时会自动切换到仍然可用的 Bot，不会让插件通知突然失去发送目标。
- 插件可以跟随当前默认 Bot，也可以单独指定内置 Bot 或其他 Bot，通知发送和插件运行会使用同一个选择。
- 通知设置中的提示和状态文字已经同步调整，可以直接看出哪些改动已生效、哪些 Bot 连接失败。

## v1.1.0.4

这一版让插件常用信息和通知设置更容易找到，也改善了不同形状插件图标的显示效果。

- 插件配置弹窗新增“通知 Bot”选择，可以直接指定这个插件使用哪个 Bot，切换后立即生效，并与系统通知页面保持一致。
- 插件菜单中的“查看历史”改名为“版本历史”，名称与实际内容更贴切。
- 插件菜单新增“项目主页”，可以直接打开该插件对应的 GitHub 仓库；没有仓库来源的插件不会显示错误入口。
- 横向、圆形和透明背景的插件 Logo 现在会按原比例完整显示，不再为了填满方框而裁掉部分内容。

## v1.1.0.3

这一版修复了插件页面几个容易看着别扭的显示问题，让图标和搜索界面更清楚。

- 没有单独图标的插件只会显示平台默认图标，不再叠着一个多余的图片占位标志。
- “官方”标识换成更像认证徽章的图标，不再只是一个普通对勾。
- 修复插件搜索中的筛选栏被插件列表压扁的问题，“全部、已安装、可更新、可安装”现在可以完整显示。
- 手机上仍然可以左右滑动搜索筛选项，同时隐藏了多余的滚动条。

## v1.1.0.2

这一版解决了插件历史日志在重启或更新后消失的问题，并让日志清理更方便管理。

- 插件历史日志会保存在日志目录中，重启平台或更新版本后仍能继续查看。
- 日志记录会同时显示日期和时间，查看以前的运行情况更清楚。
- 系统设置的“维护”页面新增日志清理设置，可以选择是否启用、每天几点执行以及保留多少条记录。
- 定时清理会同时处理平台运行日志和插件历史日志，清理过的旧记录不会在重启后重新出现。
- 只修改日志清理设置时会立即生效，不再提示需要重启平台。

## v1.1.0.1

这一版全面美化了插件页面的视觉效果，让界面更美观、交互更流畅。

- 插件卡片鼠标悬停时会轻微上浮并显示阴影，点击时有缩放反馈，视觉效果更生动。
- 所有主按钮（如"上传插件""刷新市场""保存"）改用蓝色渐变背景，更醒目也更统一。
- 出现错误的插件卡片左侧会显示红色竖线标识，一眼就能看出哪些插件有问题。
- 可更新插件的区域会用淡黄色背景和左侧标识线突出显示，不会错过更新提示。
- "官方"徽章前面加上勾选图标，更直观地表示官方认证。
- 版本更新徽章带有轻微的脉冲动画，提醒用户注意更新。
- 插件卡片右上角的三点菜单改用更现代的SVG图标，下拉菜单带有滑入动画。
- 所有警告和错误提示前面都会显示对应的图标（三角形警告或圆形感叹号），更容易识别。
- 右下角的搜索悬浮按钮鼠标悬停时会显示"搜索插件 (Ctrl+K)"提示。
- 所有按钮、标签页和卡片使用键盘导航时会显示蓝色轮廓，方便键盘用户操作。
- 插件网格布局在大屏幕上限制最大宽度，避免卡片过宽；手机上自动调整为单列显示。
- 移除了"舒适/紧凑"显示密度切换按钮，统一使用舒适模式显示，界面更简洁。

## v1.1.0

这一版修复了几个影响使用的问题，让插件更新内容更容易查看，也让系统状态首页更直观。

- 修复备份恢复功能的路径检查错误，之前会错误地拒绝恢复备份包中的顶层文件，现在可以正常恢复所有文件。
- 修复备份恢复后系统会重复执行恢复操作的问题。现在恢复成功后会自动清理临时文件，避免每次启动都重复恢复。
- 插件作者填写更新说明后，用户可以从插件卡片右下角的三点菜单打开“查看历史”，直接了解这个版本改了什么。
- 系统状态首页加入更自然的动态效果。页面加载时会平滑显示内容，账号数量、插件数量和活跃次数发生变化时也不会突然跳动。
- 插件活跃图现在可以查看每个时间段的详细次数，指向图例时会突出对应插件；账号状态或定时任务发生变化时，页面会短暂标出变化位置。
- 添加插件仓库时会统一显示为“用户名/AWBotNest-Plugins”，其他仓库名不会被加入；保存成功、重复添加或填写了内置官方仓库时，页面都会明确告诉用户结果。

## v1.0.9

这一版主要让平台更容易看懂、备份和维护。

- 运行日志现在优先显示插件的中文名称。以前看到的通常是英文代号，现在一眼就能知道是哪个插件在运行。
- 一批难懂的报错和提示已经改成白话，例如直接告诉你“端口可能被占用”或“用户账号连接失败”，不用再猜英文报错是什么意思。
- 系统设置新增“维护”页面，可以下载整个平台的备份，也可以导入以前的备份。导入时会先检查备份是否完整，并自动下载一份当前数据作为退路；真正恢复会在重启平台时进行，避免损坏正在使用的数据。
- 账号管理页面增加更清楚的状态提示，现在能区分“在线”“未登录”“已手动下线”和普通离线，也会告诉你这个账号是否已经完成过登录。
- 插件的设置页面可以更灵活地安排输入项的顺序和宽度，内容较多时不再显得杂乱。
- 插件仓库设置变得更简单，只需要填写公开仓库地址。自动检查时间由平台管理，不再需要手动填写；私有仓库也不再提供入口。
- 插件页面右下角新增搜索按钮。插件多时可以按名称、作者、说明或英文标识快速查找，并直接打开设置、安装或更新。
- 网页整体样式更加统一，系统状态是否正常会直接显示在页面顶部。插件可以按状态筛选并切换卡片大小，点击卡片还能查看完整说明、安装状态和常用操作；手机上的弹窗和底部按钮也更方便使用。

## v1.0.8

这一版让复杂插件可以拥有自己的设置页面，也让插件具备打开网页并读取内容的能力。

- 复杂插件可以自带专属设置页面，用来展示图表、运行状态和更丰富的操作。普通插件仍然使用原来的设置表单，不受影响。
- 专属设置页面可以安全地读取插件数据或执行操作，只有已经登录的管理员才能使用。
- 插件的页面文件也受到登录保护，未登录的人无法直接打开。
- 平台新增网页访问能力。插件需要读取动态网页或遇到网站访问限制时，可以直接使用，不必由每个插件重复安装浏览器。相关文件只会在第一次使用时下载并保存，不使用这项功能就不会额外占用空间。
- 新增一份专属设置页面的示例插件，插件作者可以照着修改。

## v1.0.7

这一版主要改善插件设置页面，让管理员填写配置更方便。

- 新增“列表”输入项，可以一行行添加或删除规则，不用把多条内容挤在一个输入框里。
- 新增“会话选择”输入项，需要选择群组、频道或私聊时可以直接从列表中挑选，不必到处查找很长的会话编号。账号尚未登录时也可以手动填写。
- 新增“操作按钮”，插件可以在设置页面放置“测试连接”“立即运行”等按钮，点击后马上显示结果。
- 新增“说明和状态”区域，可以显示使用提示、上次运行时间或已经处理的数量。
- 保存设置时会检查必填内容和数字范围，填写不完整或数字不合适时会直接指出具体位置。
- 插件可以保存自己的运行状态，也可以更方便地下载消息中的图片和文件。
- 修复会话尚未选择却显示数字 0，以及部分必填内容没有被正确检查的问题。

## v1.0.6

这一版新增“外部通知入口”（Webhook），其他程序可以通过专属地址向平台或插件发送消息。

- 外部程序可以向平台发送内容，再由平台把它作为通知推送给管理员。
- 插件也可以拥有自己的接收地址。外部事件到达后，插件可以处理内容并返回结果。
- 所有接收地址共用一把安全密钥，可以在“系统设置 → 通知”中一键生成。没有生成密钥时，这项功能不会开启。
- 插件停用或重新加载后，旧的接收地址会立即失效，避免消息继续送到已经停止的插件。

## v1.0.5

这一版支持同时使用多个通知机器人。

- 可以在“系统设置 → 通知”中添加一个默认机器人和多个额外机器人，并为每个插件选择由哪个机器人发送通知。没有单独选择的插件仍使用默认机器人，原有设置不用修改。
- 插件的通知和主动发送的消息都会使用所选机器人。
- 删除机器人后，原来使用它的插件会自动改回默认机器人。
- 添加或删除机器人后需要重启平台，切换插件使用的机器人则会立即生效。
- 机器人的登录密钥统一放到“通知”页面管理，不再分散在其他设置中。
- 页面中的“主人”和“所有者”统一改为“管理员”，称呼更清楚。

## v1.0.4

这一版让插件访问网络时自动使用平台设置的代理。

- 开启代理后，插件访问外部网站会自动经过代理，不需要给每个插件单独设置。本机上的服务仍会直接连接。
- 修改代理后，新启动或重新加载的插件会立即使用新设置；已经建立的长期连接需要重启平台后才会切换。
- Telegram 连接、安装插件所需组件、智能助手和插件市场现在共用同一套代理设置，行为更一致。

## v1.0.3

- 修复多个插件同时接收同一类消息时互相抢消息的问题。现在每个相关插件都能收到消息。
- 插件仍然可以主动拦下一条消息，让后面的插件不再重复处理。
- 插件可以显示自己的图标，不再全部使用平台标志。
- 修复只选择一个账号，其他账号却也跟着执行的问题。插件现在会严格按照选中的账号运行。
- 插件从市场更新后，如果正在运行会自动重新加载；没有运行的插件只更新文件，不会被自动开启。
- 修复刷新插件市场后仍看不到新版本的问题，现在刷新页面就能获取最新内容。
- 取消单个插件文件原有的 8MB 限制，带有较大资源文件的插件也能安装。
- 插件需要额外组件时，平台会在启用插件时自动检查和安装。已经安装的不会重复处理，版本不兼容时会说明原因并停止启用。
- 安装额外组件时也会使用平台代理，并在网络不可用时尽快提示失败，不再长时间卡住其他插件。
- 代理设置页面新增“下载来源”选项，默认使用国内速度较快的清华镜像；留空时使用官方来源。
- 插件额外安装的组件会保存在数据目录中，更新容器或镜像后不用重新安装。

## v1.0.2

- 插件市场和智能助手访问网络时会使用平台代理，受限网络环境下也能正常使用。
- 侧边栏新增“重启平台”按钮，修改重要设置后可以直接点击重启。
- 手机浏览器可以把平台添加到主屏幕，像普通应用一样打开。手机页面也改成更适合小屏幕的底部菜单和卡片布局。
- 网页图标改为平台标志。没有填写账号信息时，只显示一条清楚的提醒，不再连续刷出报错。
- 活跃统计只计算插件真正执行的发送、回复和编辑操作，空闲时不会再增加。插件较多时图表也会自动整理。
- 修复插件启用状态偶尔自己关闭的问题。加载失败只会显示错误，不会把插件当成手动停用。
- 插件卡片上的来源地址改为简短形式，更容易阅读。
- 登录页底部显示版权和版本号。默认账号密码只写入后台日志，不再直接展示在登录页面。
- 修复重启平台或更新镜像后，原本启用的插件全部变成未启用的问题。
- 修复通知机器人没有连接成功时，插件看似开启却收不到消息的问题。
- 修复自动更新误删本地上传或自己修改的同名插件。现在只会更新确实从插件市场安装的内容。
- 安装包含多个文件的插件时，会在全部文件下载成功后再正式替换；中途失败不会留下残缺文件。
- 加强插件下载和登录安全，防止伪造下载地址、超大文件和危险文件路径。
- 首次使用默认密码登录后，必须先设置自己的密码，才能上传插件、导入内容或修改重要设置，避免平台被默认密码接管。

## v1.0.1

- 项目中的表情符号全部改为文字，说明、日志和通知的显示更统一。
- 通知改用“通知、成功、警告、错误”等文字标记，不再依赖图标区分。
- 运行日志只保留时间、级别和内容，去掉不常用的文件名和来源栏目；插件日志仍会标明是哪个插件发出的。

## v1.0.0

首个正式版本。

- 支持安装单文件插件，一个插件对应一个功能。启用或停用插件时不需要重启平台。
- 内置插件市场，也可以添加其他插件仓库，浏览并一键安装需要的插件。安装后不会自动开启，由管理员自己决定。
- 可以在网页中登录和管理多个 Telegram 账号，并为每个插件选择在哪些账号上运行。
- 网页控制台集中提供插件管理、账号管理、运行日志、系统状态和系统设置。
- 插件产生的通知由平台统一整理，标明来自哪个账号后再发送给管理员。
- 网页控制台使用用户名和密码登录，密码会经过加密保存。
